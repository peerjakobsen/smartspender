# Categorization Learnings

User corrections for how transactions should be categorized. These rules take precedence over static pattern matching.

## How to Use

1. Check this file FIRST when categorizing transactions
2. Match against the Pattern column (case-insensitive substring match)
3. If matched, use the Category/Subcategory with confidence 1.0
4. Skip static rules for matched transactions
